📌 Principais mensagens

1. O gap maior é a UI de produção

Os 3 protótipos (Ownership Map, Aliases, Timeline) vivem só em pulse-ui/ na porta 8080 com HTML/JS vanilla. Zero código em pulse-web/ (React, port 5173). Carlos no Webmotors não consegue ver Timeline em produção.

2. pulse-api (NestJS) nunca foi proxy dos endpoints novos

Sem proxy, o React teria que falar com pulse-data direto (porta 8000) — quebra a arquitetura. Phase 2 corrige isso e prepara o terreno para auth do R1.

3. Há gaps de integridade backend que o CISO não pegou

Vazamento de abstração no Protocol (list_monitors_for_service), runbook de rotação fictício, f-string SQL no _set_tenant, hide_parameters incompleto. Phase 1 fecha em ~12h.

4. Carlos page (Timeline) é 100% inacessível

SVG sem keyboard nav, sem screen reader, severidade só por cor, time-window selector cosmético, 6 estados não tratados. Phase 5 reescreve com Chart.js + acessibilidade completa.

5. Precisamos análise antes da Phase 5 (Timeline)

O pulse-data-scientist tem que validar se MONITOR_HEALTH severity é o sinal certo pro Carlos antes de a gente reescrever o chart. Senão arrisca rework. ~2h, BLOCKING.

🚨 Gaps críticos que motivaram o plano

UI de produção inexistente
Zero código observability em pulse-web/. Carlos só vê em localhost:8080 via Python http.server.
pulse-api sem proxy
React teria que falar com pulse-data direto. Quebra arquitetura + auth surface no lugar errado.
Protocol leak (P0 que CISO não pegou)
rollup_service:293 chama método que existe só no DatadogProvider. NewRelic adapter quebra com AttributeError.
Master-key rotation é fictício
RISK-8 no backlog mas scripts/rotate_obs_master_key.py não existe. Sem runbook.
Timeline 100% inacessível
SVG sem keyboard, sem screen reader, severidade só por cor. Carlos com a11y needs não usa.
Zero integration test end-to-end
Worker→DB→API path nunca foi testado integrado. Só unit tests com mocks.
9 hardcoded hex codes
"Tokens-only" é 90% honesto. Sweep + 4 novos soft-color tokens necessários.
f-string SQL em _set_tenant
Seguro hoje (UUID validado por Pydantic), H-severity se algum dia vier de string user-controlled.
